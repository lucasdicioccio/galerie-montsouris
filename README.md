# galerie-montsouris

A personal photo gallery viewer for the desktop. Opens one or more directories of photos, lets you browse them in a tiled grid or one at a time, rate them, apply filters, zoom, and run custom scripts — all from the keyboard.

## Install

```bash
cargo install --path .
```

Or just build and run in place:

```bash
cargo build --release
./target/release/galerie-montsouris ~/Pictures/2024 ~/Pictures/2025
```

Requires a GPU (Vulkan, Metal, or OpenGL) via the `wgpu` backend. No other system dependencies.

Supported image formats: JPEG, PNG, WebP, TIFF.

## Usage

```
galerie-montsouris [OPTIONS] <path1> [path2] ...
galerie-montsouris add-to-gallery <file.galerie> <dir1> [dir2] ...
galerie-montsouris apply-filter   <spec.json> <input> <output>
galerie-montsouris embed          --namespace NAME --command "cmd %p" [--filter-file spec.json] <file.galerie>
galerie-montsouris cluster        --namespace NAME --clusters K       <file.galerie>
galerie-montsouris export         <out-dir> <path1> [path2] ...

Options:
  --save-as <file.galerie>          Save the loaded collection as a gallery file, then launch.
  --edit-gallery <file.galerie>     Open a gallery for interactive editing (repeatable).
  --init-gallery <file.galerie>     Create a new gallery file with all discovered photos, then exit.
```

Each positional `path` is either a **directory** to scan or a **`.galerie` gallery file** to load. Both can be mixed freely in a single invocation.

### Discovery mode — scan directories

```bash
galerie-montsouris ~/Pictures/2024 ~/Pictures/2025
```

Photos from all directories are loaded into a single flat collection, sorted lexicographically within each directory, in the order the directories are given.

Scanning is flat (not recursive). To include a subdirectory, pass it explicitly.

### Named galleries

A `.galerie` file is a JSON playlist of absolute photo paths. Pass it directly to load that curated set:

```bash
galerie-montsouris trip-to-rome.galerie
```

Create one from a directory scan with `--save-as`:

```bash
galerie-montsouris ~/Pictures/2024/ --save-as summer2024.galerie
```

The viewer launches normally after saving. On future runs, pass the `.galerie` file instead of the directory — photos can be renamed without losing the selection, because membership is tracked by content hash, not filename.

Use `--init-gallery` instead when you want to create a gallery file and **not** launch the viewer. It also accepts multiple directories and existing `.galerie` files as source paths, and fails if the output file already exists (safe to use in scripts):

```bash
galerie-montsouris --init-gallery all-photos.galerie ~/Pictures/2024/ ~/Pictures/2025/
galerie-montsouris --init-gallery curated.galerie existing.galerie ~/Pictures/extras/
```

To append new photos to an **existing** gallery file, use `add-to-gallery`. Photos already in the gallery are matched by absolute path or content hash and skipped; all existing ratings, filters, and annotations are preserved:

```bash
galerie-montsouris add-to-gallery all-photos.galerie ~/Pictures/2026/
```

This is the right workflow for incremental embedding: run `add-to-gallery` to register the new photos, then run `embed` — it already skips photos that have an embedding, so only the new arrivals are processed.

Gallery files are plain JSON and can be edited by hand or generated by scripts:

```json
{
  "name": "summer2024",
  "background_color": "black",
  "photos": [
    "/home/alice/Pictures/2024/DSC_0042.jpg",
    "/home/alice/Pictures/2024/DSC_0107.jpg"
  ]
}
```

The `background_color` field (`"black"`, `"gray"`, or `"white"`) is optional and overrides the user-config default when this gallery file is opened. Pressing `B` while a single gallery file is loaded saves the new color back to the file automatically.

### Interactive gallery editing

Open one or more galleries for editing while browsing:

```bash
# Edit one gallery while scanning a directory
galerie-montsouris ~/Pictures/2024/ --edit-gallery favorites.galerie

# Edit two galleries at once (the file is created empty if it doesn't exist)
galerie-montsouris ~/Pictures/2024/ --edit-gallery favorites.galerie --edit-gallery trip.galerie
```

While an `--edit-gallery` is active, the **`T`** key manages membership:

| Situation | `T` behaviour |
|-----------|---------------|
| One gallery under edit | Toggle the focused photo in/out directly; a toast confirms the change |
| Multiple galleries | Open a selector panel listing all edit-galleries |

**Selector panel controls:**

| Key | Action |
|-----|--------|
| `↑` `↓` (or `K` `J`) | Move between galleries |
| `Space` / `Enter` | Toggle the focused photo in the highlighted gallery |
| `T` / `Escape` | Close the selector |

Clicking a row in the selector also toggles membership. Changes are written to the gallery file immediately on every toggle.

**Visual indicators while editing:**

- Top bar shows an amber `✏ gallery-name` label (or `✏ N galleries` for multiple).
- Tiling mode: a small amber dot in the top-right corner marks tiles whose photo belongs to at least one edit-gallery.
- Single mode: a per-gallery membership list is drawn at the bottom-left of the photo (✓ gold = member, faint · = not a member).

## Keyboard shortcuts

### Navigation

| Key | Action |
|-----|--------|
| `←` `→` or `H` `L` | Previous / next photo |
| `↑` `↓` or `K` `J` | Move up / down one row in tiling grid |
| `PgUp` `PgDn` | Jump 10 photos back / forward |

### Tiling grid

| Key | Action |
|-----|--------|
| `+` | Zoom in — fewer tiles per page (3×3 → 2×2 → 1×1) |
| `-` | Zoom out — more tiles per page (1×1 → 2×2 → 3×3 → …) |
| `T` | Toggle between tiling and single-photo view (see note below) |
| `Escape` | Return to tiling from single-photo view; exit fullscreen if active |
| click | Open clicked photo in single-photo view |

In tiling mode the focused tile is highlighted in blue. `←`/`→` move the selection one photo at a time; `↑`/`↓` move a full row. The grid scrolls vertically when tiles don't fit the viewport, and the view always follows the selection. The grid size is remembered when switching to single-photo view and restored when switching back.

> **Note:** When `--edit-gallery` is in use, `T` is taken over by gallery editing (toggle membership / open selector). Mode switching is still available via `Escape` (→ tiling) and clicking a tile (→ single).

### Single-photo view

| Key | Action |
|-----|--------|
| `+` `-` | Zoom in / out |
| `Z` | Reset zoom to fit-to-screen |
| `1` | Zoom to 1:1 pixels (actual size) |
| scroll wheel | Zoom in / out |
| drag | Pan the image (when zoomed in) |
| right-click | Context menu: copy source path or export with filters applied |

### Photo actions

| Key | Action |
|-----|--------|
| `R` | Cycle rating: unrated → ★ → ★★ → ★★★ → ★★★★ → ★★★★★ → unrated |
| `[` | Rotate photo 90° counter-clockwise |
| `]` | Rotate photo 90° clockwise |
| `B` | Cycle viewer background: black → gray → white → black |
| `I` | Toggle per-channel RGB histogram overlay (single-photo mode only) |
| `S` | Toggle slideshow (auto-advance); drag the interval control in the top bar to change speed |
| `F` | Toggle fullscreen (borderless) mode |
| `Q` | Quit |

### Context menu (single-photo mode, right-click)

| Item | Effect |
|------|--------|
| Copy source path | Copies the original file path to the clipboard |
| Export (apply filters) & copy path | Saves a filtered PNG to a temp file and copies its path |

## Annotations

Open the annotations panel with the **`✎`** button in the top-right corner (or bind `ToggleAnnotations` to a key). The panel shows all notes attached to the focused photo and lets you add or remove them. Notes are saved into the `.galerie` file; a gallery file must be loaded to save annotations.

### Searching

The search box in the top bar filters the collection. Tokens are matched case-insensitively against note text; all tokens must match (AND logic):

```
paris sunset          → photos whose notes contain both "paris" and "sunset"
cluster:3             → photos assigned to cluster 3 by the `cluster` command
cluster:3 paris       → cluster 3 AND note contains "paris"
```

### Embedding annotations

Embeddings are numerical vector representations of photos (e.g. from a CLIP or similar model) stored inside the `.galerie` file as annotations. They enable clustering and visual similarity search.

Use the `embed` subcommand to compute and store embeddings:

```bash
galerie-montsouris embed \
  --namespace clip \
  --command "my-embedder %p" \
  gallery.galerie
```

`%p` is replaced with the absolute path to each photo. The command must print the embedding to **stdout** as either:
- raw little-endian IEEE 754 `f32` bytes, or
- a single base64-encoded line of those bytes.

Photos that already have an embedding for the given namespace are skipped, so re-running after adding new photos with `add-to-gallery` is safe and only processes the new arrivals. Use `--force` to re-embed everything.

#### Pre-processing images before embedding

Pass `--filter-file <file.json>` to apply a filter stack to each image before the embedding command sees it. The file contains a single Filter object or an array, in the same JSON format used by the `apply-filter` command. This is especially useful for scaling images down to the size the model expects:

```bash
# Scale every photo to at most 512px on its longest side before embedding
echo '{"type":"CapSize","max_px":512}' > cap512.json

galerie-montsouris embed \
  --namespace clip \
  --command "my-embedder %p" \
  --filter-file cap512.json \
  gallery.galerie
```

The original files are not modified — each photo is decoded, filtered, and written to a temporary PNG that is passed to the command in place of the original path. EXIF orientation is applied first (as in the viewer), followed by the filter stack.

Example Python/NumPy script:

```python
#!/usr/bin/env python3
import sys, numpy as np
from my_model import embed   # returns a float32 ndarray
vec = embed(sys.argv[1])
sys.stdout.buffer.write(vec.astype("<f4").tobytes())
```

### Clustering

Once embeddings are stored, assign cluster labels with the `cluster` subcommand:

```bash
galerie-montsouris cluster \
  --namespace clip \
  --clusters 8 \
  gallery.galerie
```

This runs k-means on the embeddings (cosine distance, k-means++ seeding) and writes a `ClusterAssignment` annotation to each photo. Then filter by cluster in the viewer:

```
cluster:0   → photos in cluster 0
cluster:5   → photos in cluster 5
```

### Similarity search

With embeddings stored, bind `FindSimilar` to a key to filter the tiling view to the N most similar photos to the focused one:

```toml
[[keybindings]]
key       = "F"
modifiers = ["ctrl"]
action    = "FindSimilar"
args      = { namespace = "clip", count = 20 }
```

Pressing the key replaces the current view with the 20 nearest neighbours (by cosine similarity), most similar first.

## Filters

Filters are applied non-destructively and stored in the `.galerie.json` sidecar file. Open the filter sidebar with `E` (single-photo mode) to add and adjust filters interactively.

Adjacent same-type filters are merged (e.g. two 90° rotations become one 180° rotation). Non-adjacent same-type filters are kept separate, enabling pipelines like `[Crop → Rotate → Crop]`.

| Filter | Description |
|--------|-------------|
| `Rotate` | CW rotation in degrees. Multiples of 90° are lossless; other angles use bilinear interpolation with an expanded canvas. Supports a custom centre of rotation. |
| `Flip H` / `Flip V` | Horizontal / vertical mirror. |
| `Crop` | Normalised crop rectangle (0.0–1.0 per axis). |
| `Scale` | Resize by a factor (< 1 shrinks, > 1 enlarges). |
| `Exposure` | Adjust brightness in EV stops (positive = brighter). Adjacent stops accumulate. |
| `Contrast` | Scale contrast around the midpoint (factor > 1 = more contrast). Adjacent factors multiply. |
| `Sharpen` | Unsharp-mask sharpening with a 1.5σ Gaussian. `amount` 0.5–2.0 is typical; values < 0 blur. |
| `Clarity` | Local contrast / micro-contrast via a 10σ blur. Boosts regional detail without affecting global tone. |
| `Curves` | Per-channel (R, G, B) tone curves. Each curve is a list of `[input, output]` control points (0.0–1.0) interpolated linearly. Add / remove points in the sidebar. |
| `Cap Size` | Shrink so the longest dimension is at most N pixels; no-op if already smaller. |
| `Border` | Add a solid-colour border. Thickness in pixels, colour as RGBA. |

## Persistent data

Ratings and filters are saved to a `.galerie.json` sidecar file next to the photos in each directory. The file travels with the photos if you move the directory.

```json
{
  "3f8a1c92b47de051": { "rating": 4 },
  "a0e2d71f5c983b64": { "filters": [{ "type": "Rotate", "degrees": 90 }] }
}
```

Keys are a content-derived hash (seahash of the first 128 KB of each file), so **ratings and filters survive file renames**. If you open an old sidecar that uses filename keys, it is migrated automatically on first load.

EXIF orientation (the tag cameras write when shooting in portrait) is applied automatically and is not stored — it is re-read from the file on every load.

## Configuration

On first run, copy the example below to `~/.config/galerie-montsouris/config.toml` and edit to taste. Without a config file the built-in defaults are used.

```toml
[general]
tile_count              = 9      # starting grid size (cols = ceil(sqrt(n)))
cache_size              = 50     # max images kept in GPU memory
slideshow_interval_secs = 5.0   # can also be changed live via the top-bar drag control
background_color        = "black"  # default viewer background: "black", "gray", or "white"

# Every key binding follows this shape:
[[keybindings]]
key       = "ArrowRight"   # egui::Key variant name (see list below)
modifiers = []             # any of: "ctrl", "shift", "alt"
action    = "Navigate"
args      = { direction = "next" }
```

| `ToggleHistogram` | `{}` — toggle per-channel RGB histogram overlay in single-photo mode |
| `Quit` | `{}` |
| `CycleRating` | `{ values = [1, 2, 3, 4, 5] }` |
| `ApplyFilter` | `{ filter = "RotateLeft"\|"RotateRight"\|"Rotate180" }` |

### Key names

Letters `A`–`Z`, digits `Num0`–`Num9`, `F1`–`F12`, `ArrowLeft`, `ArrowRight`, `ArrowUp`, `ArrowDown`, `Escape`, `Enter`, `Space`, `Tab`, `Backspace`, `Delete`, `Insert`, `Home`, `End`, `PageUp`, `PageDown`, `Plus`, `Minus`, `Comma`, `Period`, `Semicolon`, `Colon`, `Slash`, `Backslash`, `OpenBracket`, `CloseBracket`, `Backtick`.

### Running scripts

Scripts receive arguments built from a template list. Each element of `args` is expanded before being passed to the process. Standard output (or stderr if stdout is empty) is shown as a brief on-screen toast.

| Token | Expands to |
|-------|-----------|
| `%p` | Absolute path to the focused photo |
| `%d1` | Display width in pixels (after EXIF + user-filter rotation) |
| `%d2` | Display height in pixels |
| `%r` | Aspect ratio — `width/height`, 4 decimal places (e.g. `1.7778`) |
| `%%` | Literal `%` |

When `pass_filters_stdin = true` the script's stdin receives the photo's JSON filter stack (e.g. `[{"type":"Rotate","degrees":90}]`, or `[]` if none).

```toml
[[keybindings]]
key       = "E"
modifiers = ["ctrl"]
action    = "RunScript"
args      = { path = "~/.local/bin/export-photo.sh", args = ["%p", "--width=%d1", "--height=%d2"], pass_filters_stdin = false }
```

Omitting `args` defaults to `["%p"]` — the focused photo's path as the sole argument. A second invocation is blocked while a script is already running.

## Extending

The action and filter systems are designed to be open-ended:

- **New action**: add a variant to `Action` in `src/actions.rs`, a parsing arm in `from_binding`, and a dispatch arm in `execute_action`. Wire it to a key in the config.
- **New filter**: add a variant to `Filter` in `src/filters.rs` (serde-tagged), implement its `apply` and `merge_same` arms, add a parsing arm in `ApplyFilter`'s `from_binding` handler, and wire up the sidebar UI in `src/app.rs`.
